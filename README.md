# Pokémon Card Centering Analyzer

포켓몬 카드 사진에서 **센터링**(콘텐츠가 카드 정중앙에 있는지)을 계산한다.
왼/오른쪽, 위/아래 여백(색 테두리) 비율이 50:50에 가까울수록 좋은 등급이며,
**1% 차이로도 등급이 갈리므로 서브픽셀 수준**으로 측정한다.

투명 슬리브/탑로더, 손가락 가림, 원근 기울기, 조명 반사가 있는 실제 손사진에서
동작하도록 설계했다. 딥러닝 없이 고전 컴퓨터비전(OpenCV)만 사용한다.

## 파이프라인
1. **검출 & 정류** (`src/detect.py`)
   - GrabCut으로 카드 실루엣(코스)을 잡아 슬리브/배경/손가락을 배제.
   - 각 변의 **색 테두리 바깥 에지**를 Lab 채도(chroma) 기반으로 서브픽셀 검출하고
     RANSAC 직선 피팅 → 인접 직선 교점으로 정밀 코너 4개.
   - 원근(호모그래피) 변환으로 종횡비 63:88의 직사각형으로 정류(배경 여백 포함).
2. **테두리 검출** (`src/borders.py`)
   - 각 변에서 색 테두리의 **바깥(카드 절단면)**과 **안쪽(아트워크 경계)** 에지를
     로컬 기준색 + 색거리 임계로 서브픽셀 검출. 다수 스캔라인의 중앙값으로 안정화.
   - 테두리 색은 4변 공통이므로, 배경에 오염된 변은 교차 합의색으로 재측정.
3. **센터링 & 등급** (`src/centering.py`)
   - L/R, T/B 비율, 최악축 기준 PSA 스타일 등급, 검출 편차 기반 신뢰도(high/medium/low).
4. **시각화** (`src/visualize.py`)
   - 정류 카드에 바깥(초록)/안쪽(빨강) 테두리, 각 변 폭, L:R·T:B 게이지, 요약 패널.

## 설치
```bash
conda env create -f environment.yml   # 'card' 환경 생성 (또는 conda create -n card python=3.10)
conda activate card
pip install opencv-python numpy scipy matplotlib Pillow
```

## 실행
```bash
# 단일 이미지 → output/<name>_report.png + <name>.json
python run.py data/front001.jpg

# 재현성 + 조명 강건성 검증 (front끼리 / back끼리 일치 확인)
python validate.py
```

프로그램적 사용:
```python
from src import pipeline
res = pipeline.analyze("data/front001.jpg")
print(res.to_dict())        # 앱/웹앱에서 그대로 소비 가능한 구조
```

## 반자동 웹앱 (권장 — 모든 테두리 색 지원)
임의 테두리 색(노랑/파랑/회색/금색)에서 자동검출은 강건성이 부족하므로, **사람이 확정하는 반자동 3단계 웹앱**을 제공한다. 자동 추천은 참고용이고 최종 판단은 사용자가 한다. UI는 영어.

**정적 호스팅 (GitHub Pages) — 백엔드 불필요:** 원근 변환을 브라우저(JS 호모그래피)에서 수행하므로 `index.html` 하나만으로 완전히 클라이언트에서 동작한다. 저장소 루트의 `index.html`을 Pages로 서비스하면 된다. (백엔드가 없을 때는 시작 박스에서 네 꼭짓점을 직접 맞춘다.)

```
# 로컬 정적 실행 (백엔드 없이)
python -m http.server 8000   # http://localhost:8000

# 로컬 + 자동 코너 추천(선택, GrabCut) — FastAPI 백엔드
conda activate card
python -m uvicorn webapp.server:app --port 8000   # http://localhost:8000
```
백엔드가 켜져 있으면 업로드 시 카드 네 모서리를 자동 추천하고, 없으면(예: Pages) 기본 박스에서 수동으로 맞춘다. 어느 쪽이든 원근 변환·측정은 동일하게 동작.

3단계 (1×3 그리드):
1. **바깥 꼭짓점** — 업로드하면 카드 네 모서리를 자동 추천. 핸들을 드래그해 정확히 맞춘다(원근/기울어진 사진 OK, 드래그 시 **돋보기**). "정류하기"로 직사각형 변환.
2. **안/바깥 테두리** — 정류된 카드에서 파란(바깥·카드 끝)·빨강(안쪽·아트 경계) 선을 드래그(돋보기 제공). 추천값이 미리 채워짐.
3. **센터링 결과(실시간)** — ②의 드래그에 따라 좌/우·상/하 비율, 여백 px, 등급이 즉시 갱신.

백엔드(`webapp/server.py`)는 `src/`의 검출·원근변환·테두리추천을 재사용하고, 측정은 브라우저에서 계산한다.

## 정확도 (자동 파이프라인, 유채색 테두리)
- **좌/우 센터링**: 같은 카드의 여러 사진에서 표준편차 ~0.2–0.6%p (매우 안정적).
- **상/하 센터링**: 대부분 안정적이나, 짧은 변(상/하)이 밝은 배경/글레어에 인접한
  일부 사진에서 Stage-1 정류 오차로 편차가 큼 → **신뢰도 low/medium 으로 자동 표시**.
  여러 장을 찍어 중앙값을 쓰면(=`validate.py` 방식) 실제 센터링을 안정적으로 얻는다.
- 밝기/감마 변화에 대체로 강건(밝기 ±30% 시 <0.4%p 변동).
